从 backgroundRaymarch.ts 移除平台 SDF 逻辑，删除 sdfPlatforms.ts 和 triplanarMapping.ts。

## Tasks

### 1. 修改 backgroundRaymarch.ts 的 map() 函数

**File:** `.worktrees/phase-01-sdf-background/frontend/src/components/tree/shaders/backgroundRaymarch.ts`

**Changes:**
- 移除 `import { SDF_PLATFORMS } from './sdfPlatforms'` (line 3)
- 移除 `import { TRIPLANAR_MAPPING } from './triplanarMapping'` (line 4)
- 移除 shader 字符串中的 `${SDF_PLATFORMS}` 插入 (line 31)
- 移除 shader 字符串中的 `${TRIPLANAR_MAPPING}` 插入 (line 23)

### 2. 适配 map() 返回值变更

map() 从 `vec2` 改为 `float` 后，以下调用点需要适配：

- `calcNormal()`: `map(p + k.xyy * eps).x` → `map(p + k.xyy * eps)`
- `calcAO()`: `map(p + h * n).x` → `map(p + h * n)`
- `softShadow()`: `map(ro + rd * t).x` → `map(ro + rd * t)`
- `main()` 中的 raymarch 循环: `vec2 res = map(p); float d = res.x;` → `float d = map(p);`
- 移除 `hitMaterialID` 变量和 `isPlatform` 分支

### 3. 简化 main() 中的着色逻辑

移除 `isPlatform` 分支后，main() 的 hit 处理只保留 vista 着色：
- 移除 `float hitMaterialID = 0.0;` 和 `hitMaterialID = res.y;`
- 移除整个 `if (isPlatform) { ... }` 块（lines 190-229）
- 保留 `else` 块的内容作为唯一的着色路径（vista 着色）
