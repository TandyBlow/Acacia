New map() function — composited with platform SDF:
```glsl
float map(vec3 p) {
  int style = int(uStyleType + 0.5);

  // Vista SDF dispatch (existing)
  float vistaD;
  if (style == 1) vistaD = mapSakura(p);
  else if (style == 2) vistaD = mapCyberpunk(p);
  else if (style == 3) vistaD = mapInk(p);
  else vistaD = mapDefault(p);

  // Platform SDF (NEW — PLAT-03)
  // Platform origin: placed uPlatformZ units ahead of camera along forward vector
  vec3 ro = vec3(0.0, uCamY, uCamZ);
  vec3 forward = normalize(vec3(0.0, sin(uCamPitch), cos(uCamPitch)));
  vec3 platformOrigin = ro + forward * uPlatformZ;
  float platformD = sdPlatform(p - platformOrigin, int(uPlatformType + 0.5));

  return min(vistaD, platformD);
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| uPlatformType / uPlatformZ → GPU (CPU to Shader) | Untrusted uniform values cross from TypeScript runtime into the GPU fragment shader via THREE.IUniform. Malformed values could cause shader undefined behavior (e.g., type=-1 accessing non-existent array index in a switch) or NaN/Inf propagation. |
| uPlatformZ → raymarch (GPU internal) | Extreme uPlatformZ values (<2 or >5) could place the platform outside the visible range or clip into the camera. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04-01 | Tampering | uPlatformType uniform — out-of-bounds type value (e.g., -1, 5, 999) | mitigate | (1) SdfParamRegistry enforces min=0, max=4 on the TypeScript side. (2) sdPlatform dispatch returns 1e10 for unknown types (safe miss). (3) applyParamsToUniforms uses Math.round() for int uniforms to ensure integer values. Combined: even if bounds are bypassed, the GLSL fallback ensures no geometry is rendered for invalid types. |
| T-02-04-02 | Tampering | uPlatformZ uniform — out-of-bounds value (e.g., 0, 100, negative) | mitigate | SdfParamRegistry enforces min=2, max=5. A value of 0 would place the platform at camera origin (invisible, inside the near plane). A value of 100 would place it beyond uFogDistance (invisible, fogged out). Neither causes undefined behavior. |
| T-02-04-03 | Denial of Service | Shader recompilation with SDF_PLATFORMS (increased shader complexity) | accept | The additional GLSL code (~300 lines) increases shader compilation time marginally. ERR-01 fallback to sky2d.ts (Phase 8) handles catastrophic compilation failures. Phase 2 assumes compilation succeeds — validated by the production build check in Task 2. |
| T-02-04-04 | Denial of Service | Platform SDF evaluation per raymarch step (performance regression) | accept | Each platform type evaluates 5-15 primitive SDFs + up to 7 detail evaluations per step. At 80 steps max, this adds ~800-1760 additional SDF operations per pixel. However, platform-hit pixels terminate early (t < ~8, ~10 steps), reducing average cost. Full performance optimization is deferred to Phase 8 (ARCH-04: skip platform evaluation when t > uPlatformZ + 3.0). |
