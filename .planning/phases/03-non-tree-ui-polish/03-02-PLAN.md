    ```python
    def get_quiz_trends_sqlite(owner_id: str, conn: sqlite3.Connection, days: int = 30) -> dict:
        """
        Get daily quiz trend data: avg mastery score and accuracy per day for the last N days.
        Joins quiz_records with nodes to access mastery_score.
        Uses parameterized queries exclusively.
        """
        rows = conn.execute(
            """SELECT
                 date(qr.answered_at) as day,
                 AVG(CASE WHEN qr.is_correct = 1 THEN nodes.mastery_score END) as avg_mastery,
                 AVG(CASE WHEN qr.is_correct = 1 THEN 1.0 ELSE 0.0 END) as accuracy,
                 COUNT(*) as review_count
               FROM quiz_records qr
               JOIN nodes ON qr.node_id = nodes.id
               WHERE qr.owner_id = ? AND qr.answered_at >= date('now', ?)
               GROUP BY day
               ORDER BY day ASC""",
            (owner_id, f'-{days} days'),
        ).fetchall()

        trends = []
        for r in rows:
            from datetime import datetime
            dt = datetime.strptime(r["day"], "%Y-%m-%d")
            trends.append({
                "date": r["day"],
                "date_epoch": int(dt.timestamp()),
                "avg_mastery": round(float(r["avg_mastery"] or 0), 4),
                "accuracy": round(float(r["accuracy"] or 0), 4),
                "review_count": int(r["review_count"] or 0),
            })

        return {"trends": trends}
    ```

    IMPORTANT details:
    - Use `date('now', ?)` with the parameter `f'-{days} days'` as the second `?` value (e.g., `'-30 days'`) -- this is SQLite's built-in date arithmetic. The `days` parameter from the endpoint is validated before constructing the `f'-{N} days'` string for the parameterized query.
    - The SQL uses `?` placeholders exclusively (parameterized). The `f'-{days} days'` is constructed in Python and passed as a bound parameter value, NOT interpolated into the SQL string.
    - The `JOIN nodes ON qr.node_id = nodes.id` requires that `node_id` in quiz_records references a valid nodes row. If no matching node exists, those quiz_records rows are excluded from the aggregation (LEFT JOIN intentional because inner logic: if a quiz_record has no corresponding node, the mastery_score is unknown).
    - `round(float(x or 0), 4)` pattern matches the existing code in `get_quiz_stats_sqlite` at lines 521-524.
    - `int(dt.timestamp())` returns Unix epoch seconds for the date at midnight UTC.

    ```python
    def get_review_heatmap_sqlite(owner_id: str, conn: sqlite3.Connection, days: int = 365) -> dict:
        """
        Get daily review counts for the calendar heatmap.
        Queries quiz_records.answered_at grouped by date.
        Uses parameterized queries exclusively.
        """
        rows = conn.execute(
            """SELECT
                 date(answered_at) as day,
                 COUNT(*) as count
               FROM quiz_records
               WHERE owner_id = ? AND answered_at >= date('now', ?)
               GROUP BY day
               ORDER BY day ASC""",
            (owner_id, f'-{days} days'),
        ).fetchall()

        return {
            "days": [
                {"date": r["day"], "count": int(r["count"])}
                for r in rows
            ]
        }
    ```

    IMPORTANT:
    - The query aggregates ALL quiz_records for the user, grouped by calendar day. Each row is one day with the count of quiz records answered that day.
    - `WHERE owner_id = ?` enforces access control (ASVS 4.1.2) -- only the authenticated user's own data is returned.
    - `answered_at >= date('now', ?)` with parameter `f'-{days} days'` uses SQLite date arithmetic. The `f'-{N} days'` is a bound parameter value, not SQL interpolation.
    - Return format is `{"days": [{"date": "2026-04-30", "count": 5}, ...]}` -- array of date+count objects, sorted ascending by date.
