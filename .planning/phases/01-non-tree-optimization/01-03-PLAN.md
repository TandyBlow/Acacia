**Architecture requirements (per CONTEXT.md decisions D-16, D-17):**

1. **Pure ASGI class** — NOT BaseHTTPMiddleware. Uses `__init__(self, app)` and `async __call__(self, scope, receive, send)`.

2. **Request fields to log:** `method`, `path`, `status_code` (from response), `duration_ms` (float, rounded to 2 decimal places), `user_id` (string).
   - If the request has an Authorization: Bearer header, extract the user_id by calling `verify_token(token)`. If the token is valid, use `payload["sub"]` as user_id. If the token is missing or invalid, use "anonymous".
   - Duration is measured from entry into `__call__` to when the response completes (wrapped send is called with `http.response.body`).

3. **Structured key=value log format (per D-17):**
   ```python
   logger.info(
       "method=%s path=%s status=%s duration_ms=%s user=%s",
       method, path, response_status, duration_ms, user_id,
   )
   ```
   This produces output like: `2026-04-29T10:30:00 INFO acacia.request method=GET path=/nodes/123 status=200 duration_ms=12.34 user=abc-123`

4. **Logging configuration:**
   - Logger name: `acacia.request`
   - Use Python stdlib `logging` module (no external dependencies)
   - Configure the logger with a StreamHandler at module level
   - Use a Formatter that prepends timestamp and level: `logging.Formatter("%(asctime)s %(levelname)s %(name)s %(message)s")`

5. **Response wrapping:** Wrap the `send` callable to capture `response_status` from `http.response.start` messages. The wrapped send must intercept the status BEFORE passing the message to the real send.

6. **Non-HTTP scopes:** If `scope["type"] != "http"`, pass through to `self.app(scope, receive, send)` without logging.

7. **Best-effort user extraction:** The RESEARCH.md open question #3 recommends heuristic JWT extraction — decode the payload without full cryptographic verification for logging purposes. ACCEPT this recommendation: parse the Bearer token from the Authorization header, call `verify_token(token)` which returns None for invalid tokens. If None, use "anonymous". This is a best-effort identification for logging, not a security gate.

**Class structure:**
```python
class LoggingMiddleware:
    def __init__(self, app):
        self.app = app

    async def __call__(self, scope, receive, send):
        if scope["type"] != "http":
            await self.app(scope, receive, send)
            return

        start_time = time.time()
        method = scope.get("method", "UNKNOWN")
        path = scope.get("path", "/")
        user_id = self._extract_user_id(scope)
        response_status = 0

        async def wrapped_send(message):
            nonlocal response_status
            if message["type"] == "http.response.start":
                response_status = message["status"]
            await send(message)

        try:
            await self.app(scope, receive, wrapped_send)
        finally:
            duration_ms = round((time.time() - start_time) * 1000, 2)
            logger.info(
                "method=%s path=%s status=%s duration_ms=%s user=%s",
                method, path, response_status, duration_ms, user_id,
            )

    def _extract_user_id(self, scope) -> str:
        # Parse Authorization header from scope["headers"]
        # headers is a list of (bytes_key, bytes_value) tuples
        for header_name, header_value in scope.get("headers", []):
            if header_name == b"authorization":
                token_str = header_value.decode("utf-8", errors="replace")
                if token_str.startswith("Bearer "):
                    token = token_str[7:]
                    payload = verify_token(token)
                    if payload and "sub" in payload:
                        return payload["sub"]
                break
        return "anonymous"
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP request → LoggingMiddleware | Request metadata (method, path, headers) is read for logging |
| Authorization header → LoggingMiddleware | JWT token parsed for best-effort user identification |
| LoggingMiddleware → stdout/stderr | Log output may be captured by systemd journal, Nginx logs, or terminal |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-03-01 | Information Disclosure | LoggingMiddleware | mitigate | Only logs method, path, status, duration_ms, and user_id. Does NOT log request bodies, query parameters, headers (except for token extraction which is not logged), or response bodies. User ID from JWT is already a non-secret identifier. |
| T-01-03-02 | Elevation of Privilege | main.py middleware order | mitigate | The plan explicitly specifies and verifies the middleware registration order: LoggingMiddleware → RateLimitMiddleware → CORSMiddleware. If order were reversed (rate-limit inside logging), rate-limited requests wouldn't be logged. The grep verification in accept criteria ensures correct order. |
| T-01-03-03 | Denial of Service | LoggingMiddleware | accept | The logging middleware performs minimal work per request (time capture, header scan, string formatting). No I/O beyond the final log write which is async in uvicorn's logging handler. Acceptable overhead for debugability benefit. |
| T-01-03-04 | Information Disclosure | verify_token call in middleware | mitigate | verify_token() returns None for invalid/expired tokens without logging the error. The middleware treats all token failures identically (falls back to "anonymous"). No differentiation between "no token", "expired token", and "invalid token" in logs — prevents information leakage about token validity. |
</threat_model>
