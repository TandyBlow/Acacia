Acacia 项目 Supabase → SQLite 迁移 对话记录
=====================================================
日期：2026-04-26

一、背景与决策
------------

用户想把 Acacia 项目从 Supabase（托管 PostgreSQL + Auth）迁移到自己的 VPS 上自托管。

VPS 配置：SPECIAL 20G KVM PROMO V5 - CN2 GIA ECOMMERCE
- RAM: 1GB（加 2GB swap 后约 3GB 可用）
- 存储: 20GB SSD
- 带宽: 1TB/月
- 线路: CN2 GIA（回国速度优秀）

关键决策：
1. 用 SQLite 替代 PostgreSQL（省 200MB RAM，笔记 App 读写比极高，SQLite 完全胜任）
2. 不跑 AI 模型在本地，所有 AI 功能调外部 API（Claude/GPT），VPS 只做 HTTP 转发
3. 前端也放 VPS，用 Nginx 服务静态文件 + 反代 API（不用 Vercel，国内访问差）
4. 不加 Cloudflare（免费版回国线路不优化）

并发能力估算（1GB RAM + SQLite 方案）：
- 同时点击加载树: ~70 并发
- 活跃在线: ~300-500
- DAU: ~800-1500
- 成长路径: 0-500 DAU 够用，500-1500 调优，1500+ 加一台最便宜机器


二、已完成的代码改动
------------------

备份: git tag pre-supabase-removal（回退命令: git checkout pre-supabase-removal）

### 后端改动

1. **database.py** — 添加 depth 列 + WAL 模式优化
   - nodes 表新增 depth INTEGER NOT NULL DEFAULT 0
   - 添加迁移: ALTER TABLE nodes ADD COLUMN depth ...
   - WAL 模式: PRAGMA journal_mode=WAL, synchronous=NORMAL, cache_size=-2000, temp_store=MEMORY

2. **ai_generate_service_sqlite.py** — 新建
   - 从 ai_generate_service.py 迁移纯函数: call_llm, parse_llm_json, find_parent_match, SYSTEM_PROMPT
   - 新增 ai_generate_nodes_sqlite(user_input, owner_id, conn) 使用 SQLite
   - content 存储为纯文本（非 JSON {markdown: ...}）

3. **tag_service_sqlite.py** — 重写
   - 吸收 tag_node + DOMAIN_KEYWORDS（原来从 tag_service.py 导入）
   - tag_all_nodes_sqlite 改为单事务，不再每次更新开新连接

4. **style_service_sqlite.py** — 重写
   - 吸收 STYLE_RULES（原来从 style_service.py 导入）

5. **quiz_service_sqlite.py** — 重写
   - 吸收 call_llm_for_quiz, parse_quiz_json, LLM 配置变量（原来从 quiz_service.py 导入）

6. **main.py** — 大幅精简
   - 删除所有 Supabase 端点（/generate-tree/{user_id}, /tag-nodes/{user_id}, /style/{user_id}, /ai-generate-nodes/{user_id}, /ai-generate-question/{user_id}/{node_id}, /submit-answer-supabase/{user_id}/{node_id}, /quiz-stats/{user_id}）
   - /local/ 前缀端点重命名为标准路径:
     - /local/generate-tree-skeleton → /generate-tree-skeleton
     - /local/tag-nodes → /tag-nodes
     - /local/style → /style
   - 新增 JWT 认证端点:
     - POST /ai-generate-nodes（原 Supabase 版无认证）
     - GET /quiz-stats（替代 /quiz-stats-local 和 /quiz-stats/{user_id}）
   - 所有端点现在都要求 JWT 认证（通过 Depends(get_current_user)）

7. **删除的文件**: db.py, tree_repository.py, tag_service.py, style_service.py, quiz_service.py, ai_generate_service.py, persistence.py, tree_generator.py, test_tag_style.py, test_tree_generator_integration.py, EXAMPLES.md, TREE_GENERATOR.md

8. **requirements.txt** — 移除 supabase 依赖

9. **deploy_to_hf.py** — 更新提示信息（SUPABASE_URL/SERVICE_KEY → JWT_SECRET/SILICONFLOW_API_KEY）

10. **migrate_from_supabase.py** — 新建，一次性数据迁移脚本

四、下一步（待完成）
------------------

1. **git push** — 推送代码到远程仓库

2. **VPS 部署** — SSH 登录服务器后执行:
   ```bash
   git clone https://github.com/TandyBlow/Acacia.git /opt/acacia
   sudo bash /opt/acacia/deploy/setup.sh
   sudo nano /etc/systemd/system/acacia.service  # 设置 JWT_SECRET 和 SILICONFLOW_API_KEY
   sudo systemctl daemon-reload
   sudo systemctl restart acacia
   sudo systemctl status acacia
   ```

3. **验证** — 浏览器访问服务器 IP，注册、创建节点、编辑、移动、删除

4. **HTTPS** — `sudo certbot --nginx -d 你的域名.com`

5. **数据迁移**（如果需要保留 Supabase 现有数据）:
   ```bash
   cd /opt/acacia/backend
   pip install supabase  # 临时安装
   SUPABASE_URL=xxx SUPABASE_SERVICE_KEY=xxx python migrate_from_supabase.py
   pip uninstall supabase  # 用完卸载
   ```
   注意：用户需要重新注册（Supabase auth 用户的密码哈希不可直接导出）

6. **Swap 扩容** — 建议在 VPS 上加 2GB swap:
   ```bash
   sudo fallocate -l 2G /swapfile
   sudo chmod 600 /swapfile
   sudo mkswap /swapfile
   sudo swapon /swapfile
   echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab
   echo 'vm.swappiness=10' | sudo tee -a /etc/sysctl.conf
   sudo sysctl -p
   ```

7. **.env 文件注意** — 前端 .env 和 .env.production 包含旧的 Supabase 凭证，已替换但需要确认 .gitignore 中已排除 .env
